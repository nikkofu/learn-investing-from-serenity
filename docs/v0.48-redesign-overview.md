## 1. 背景与目标

当前产品（v0.47.1）在**功能纵深**上已经很扎实：19 个功能页、5 套主题 × 明暗双模、TradingView 级 Pro 画布、回测证明引擎、套利校准、纸面交易等。但**信息架构与视觉系统**没有随功能膨胀而升级，出现明显的「工具堆叠感」：

- 顶栏 **17 个扁平入口**横向平铺，无分组、无层级，窄屏换行成两三行，挤占首屏。
- 缺少全局**搜索 / 命令面板**——19 个页面只能靠肉眼在长导航里找。
- **首页信息密度低**：静态 Hero + 五因子 + 六步工作流 + 知识库 teaser，不承载「今日该看什么」的仪表盘价值。
- **页面头部不统一**：各页自行手写标题/副标题/操作区，缺少 `PageHeader`、面包屑、统一的空状态。
- **排版未针对中文优化**：字体栈是 Geist（仅 latin），中文走系统回退，缺乏统一字阶 / 数字等宽规范。
- 圆角 / 间距 / 阴影**散用**（`rounded-md/lg/xl/2xl` 混用），无 token 约束，视觉一致性靠手感。

### 目标（v0.48 北极星）
把「一堆专业工具」整合为**一个有层次、可被快速导航、视觉国际化的投研工作台**：

1. **信息架构**：17 扁平入口 → 5 大分组的**可折叠侧边导航** + 精简顶栏（含全局搜索/命令面板 + 面包屑）。
2. **首页**：静态介绍页 → **真实仪表盘**（市场快照 + 我的自选 + 今日热门 + 最近告警 + 快捷研究入口 + 知识库）。
3. **全局视觉**：在现有 token 体系上补齐**字体 / 间距 / 圆角 / 阴影 / 字阶** token，统一 `PageHeader`、`Card`、`KPI`、`Badge`、`Button`、`EmptyState` 等基础组件，达到 Linear / TradingView / Bloomberg Terminal 级别的克制、专业、国际化观感。

## 2. 设计原则（贯穿全程）

| # | 原则 | 落地约束 |
|---|------|----------|
| P1 | **复用而非推翻** | 现有 5 主题 × 明暗的 CSS 变量体系是优秀地基，**只扩展不重写**；所有新组件只消费语义 token（`var(--accent)` 等），不写死颜色。|
| P2 | **专业克制的国际化审美** | 深色优先、低饱和大面积底色 + 高饱和强调色点缀；信息密度高但留白节奏清晰；圆角统一、阴影克制；动效短促（150–250ms）、仅用于反馈。|
| P3 | **数据优先** | 数字一律 `tabular-nums` 对齐；涨跌沿用 A 股惯例（红涨绿跌，且**不随主题色变**）；表格可排序 / 可过滤 / 粘性表头（沿用 v0.46 回测表范式推广全站）。|
| P4 | **导航可达性** | 任意页面 ≤ 2 次点击可达；全局搜索/命令面板（`Cmd/Ctrl+K`）可跳任意页与个股；面包屑标识层级。|
| P5 | **响应式** | 侧边栏在窄屏折叠为抽屉（drawer）+ 顶栏汉堡；首页栅格 1→2→3 列自适应。|
| P6 | **零破坏 / 零口径漂移** | 不改任何回测 / 套利 / 打分**计算口径**；不删功能；免责文案继续走 `disclaimers.ts`；A 股股票池约束（`universe.ts`）不动。|
| P7 | **零 / 轻新依赖** | 优先纯 CSS + 现有 lightweight-charts；如确需引入（如图标库 `lucide-react`、字体），单独评审，选发布满 7 天的稳定版。|
| P8 | **可访问性（a11y）** | 焦点态可见（`:focus-visible` 环）、可键盘操作、`aria-*` 标注、对比度达 WCAG AA。|

## 3. 现状盘点（研究结论）

### 3.1 技术栈
- Next.js **16.2.9（Turbopack，App Router）**、React **19.2.4**、Tailwind **v4**（`@import "tailwindcss"` + `@theme inline`）、lightweight-charts **5.2.0**、TypeScript 5。
- ⚠️ 本仓库 `AGENTS.md` 标注：这是「**改版过的 Next.js**」，实现前需先读 `node_modules/next/dist/docs/` 的对应指南，留意弃用提示。

### 3.2 布局与导航（`src/app/layout.tsx` / `src/components/Nav.tsx`）
- `layout.tsx`：`<Nav/>` 顶栏 + `<main>` 全宽内容 + `<footer>` 免责声明；根节点 `data-theme` / `data-mode` 注入 + 防闪烁内联脚本。
- `Nav.tsx`：水平 sticky 顶栏，**17 个 `<Link>` 扁平平铺** + 右侧 `ModelSelector` / `ThemeSwitcher` / `ModeToggle`。无分组、无图标、无搜索、无折叠、无面包屑。

### 3.3 设计系统（`src/app/globals.css`，457 行）
- **优点**：5 套主题（松针寒绿 / 极光靛蓝 / 日落珊瑚 / 古铜暖金 / 黑曜极客）× 明暗双模，全套语义 token：`--bg/--surface/--panel/--border/--hover/--text/--muted/--faint/--accent(-strong/-soft/-line/-fg)/--warn(...)`；热力图块 / 错误条随明暗自适应；自定义滚动条；全局指针光标。
- **缺口**：无 `--radius-*` / `--space-*` / `--shadow-*` / `--font-size-*` token；`--bg-gradient`、`--card-blur` 仅个别主题定义；中文字体未进字体栈（`--font-geist-sans` 仅 latin）。

### 3.4 页面清单（19 页，行数见括号，反映复杂度）
- 发现类：`/`(90) `/methodology`(249) `/map`(384) `/sectors`(949) `/scanner`(890) `/mining`(750)

## 4. 范围（In / Out）

**In（v0.48 做）**
- 布局外壳重构（侧边栏 + 顶栏 + 命令面板 + 面包屑），响应式。
- 菜单 / IA 重新分组（见 IA 文档）。
- 设计 token 扩展 + 基础组件库（见设计系统文档）。
- 首页改版为仪表盘（见首页文档）。
- 把 `PageHeader` / `Card` 等推广到各页**页头与容器**（渐进式，不重写页内业务逻辑）。

**Out（v0.48 不做，避免范围蔓延）**
- 不改任何**计算口径**（回测 / 套利 / 动量 / 打分 / 复权）。
- 不重写各页**业务逻辑与数据流**（只换页头/容器/视觉壳）。
- 不动 `universe.ts` 股票池约束、`disclaimers.ts` 文案口径。
- 不引入后端/数据模型变更。
- 不做 i18n 多语言文案切换（「国际化审美」指视觉语言，不是 i18n 文本；若需要英文版另立版本）。
