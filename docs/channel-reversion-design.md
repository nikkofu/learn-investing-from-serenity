# 回归通道均值回归 V1（Channel Reversion）设计文档

## 0. 背景：为什么 V5 会「错过通道下轨支撑 / 空翻多」这类买点

用户在 `/chart` 观察到：V5 跳过了图上「回归通道上升趋势下轨支撑反弹」「空翻多」等明显的低吸买点。

排查结论：**这是策略定位问题，不是技术指标算不出来。**

- V3/V4/V5 都是 **RSI 趋势跟随系统**，只有两个买入触发（`buildCardwellV4Ctx.entry`）：
  1. **区间切换上冲**：`RSI(14)` 上穿 55 且收盘站上 MA20；
  2. **正向反转/底背离**：价创近 20 根新低但 RSI 抬升。
- V5 还叠了 **ADX(14) ≥ 20 趋势闸门**（没趋势不做）与 **再入场质量闸门**。
- 「回归通道下轨支撑反弹」本质是 **均值回归 / 震荡低吸**——发生时 ADX 通常还没起来，正好被趋势闸门滤掉；这与趋势跟随在理念上**对立**。
- 因此 V5 **结构性地**捕捉不到这类点：既没有「离通道下轨多近就买」的逻辑，其趋势定位也天然排斥低吸。

**把低吸硬塞进 V5 会互相打架**（趋势闸门 vs 低吸），所以新增一个**独立、互补**的均值回归策略，而非改 V5。

## 1. 通道口径：与图上完全一致

复用 `computeRegressionChannel(candles, len=60, k=1.5)`——与 `/chart` 粉色回归通道、`/scanner` 展开评估同口径：

- 最近 60 根收盘价做最小二乘线性回归得**中轨** `mid = slope·(len-1) + intercept`；
- **上/下轨** = `mid ± 1.5σ`（σ 为拟合残差总体标准差 `/len`）；
- 滚动计算：每根 i 用「截至 i 的 60 根」拟合，得该根的 mid/upper/lower/slope，供逐根回测使用（图上只画最新一根，本策略需要每根）。

## 2. 入场：只在「上升通道」回踩下轨支撑低吸

同时满足：

1. **上升通道闸门**：相对斜率 `relSlope = slope/mid ≥ minSlopeRel(0.0008)`——只做上升通道，**不接下降通道的刀**（对应用户说的「**上升**趋势下轨支撑」）；
2. **长期均线闸门**：`close > MA60`（`requireMaTrend`）——长期趋势向下时不低吸；
3. **回踩下轨**：昨日最低价贴近/跌破下轨（`low[i-1] ≤ lower[i-1]·(1+touchPct)`，touchPct=1%）；
4. **企稳收阳**：今日 `close > 下轨` 且 `close > 昨收`；
5. **RSI 拐头**：`RSI[i] ≥ RSI[i-1]`；
6. **处通道下半区**：`close < mid`（只在下半区低吸，不在上轨附近追）。

离场后冷却 3 根。

## 3. 出场：中/上轨分批止盈 + 破位/保本/超时止损（复用引擎分批平仓）

- **中轨止盈**：触及中轨 → 卖 **1/2**，并把止损**上移到成本（保本）**，本笔立于不败；
- **上轨止盈**：触及上轨 → 剩余**全部**止盈离场；
- **通道破位止损**：收盘跌破下轨逾 `stopBufferPct(1.5%)` → 全部离场（通道支撑失守）；
- **保本止损**：中轨落袋后止损=成本，回踩即无损离场；
- **超时失效**：持有满 `maxHold(40)` 根仍未触中轨/上轨 → 离场释放资金（均值回归未兑现）。

分批平仓复用 `executeTradesNextOpen` 的 `sizePct`（与 V5 同一引擎，含 `cash+shares*close` 盯市修复）。

## 4. 回测（12 只 A 股 · 400 根日线 · 次日开盘撮合 · 双边手续费）

| 指标 | Channel Reversion | V5 | V3 | 买入持有 |
|---|---|---|---|---|
| 平均收益 | **−1.8%** | +22.1% | +71.4% | +163.7% |
| 平均最大回撤 | **−12.4%** | −11.4% | −21.1% | — |
| 平均胜率 | **52.8%（最高）** | 38.8% | 35.0% | — |
| 平均交易数 | 3.6 | 3.8 | — | — |

**诚实结论**

- 本策略**如设计所愿**：捕捉到了 V5 结构性错过的「通道下轨支撑反弹」低吸点，且**胜率全场最高（~53%）、回撤与 V5 相当**——「多数小赢」的均值回归特征。
- 但在这一篮子**大市值强趋势票**上，**平均收益约为零（−1.8%）**：均值回归在强单边趋势里会**过早止盈**（如 601869 只吃到 +18% vs 买入持有 +1826%），且偶有上升通道回踩后转为破位（002594/300024）造成亏损。
- **它的价值是「状态互补」而非「独立跑赢」**：趋势跟随（V3/V4/V5）吃单边大行情，均值回归吃**震荡/箱体**行情。本篮子恰好几乎全是趋势票、缺震荡样本，故均值回归在此表现平平；在真正箱体震荡的标的上它才会体现优势。
- 调参（`stopBufferPct` 1.5%、`minSlopeRel` 0.0008、`requireMaTrend`）已把「接刀」风险压到最低；参数在该篮子择优、存在过拟合风险。

## 5. 定位与使用建议

- **默认 Pro 策略仍为 V3**，不变。
- Channel Reversion V1 作为 **`/chart` 策略图层、`/analyze`、`/backtest/strategy` 可选变体**，用于：
  - 想抓「上升通道回踩下轨」低吸、要求高胜率/平滑净值时；
  - 判断标的处**箱体/震荡**而非单边趋势时（与趋势跟随策略互补搭配）。
- `id`: `channel-reversion-v1`；`/chart` 策略图层可叠加「下轨交易计划色块 + 买卖翻转标记」。

## 6. 关键参数（`CHANNEL_REVERSION_DEFAULTS`）

| 参数 | 默认 | 含义 |
|---|---|---|
| channelLen | 60 | 回归通道回看根数（与图上一致） |
| channelK | 1.5 | 上下轨标准差倍数（与图上一致） |
| minSlopeRel | 0.0008 | 相对斜率下限：低于则不入场（只做上升通道） |
| requireMaTrend | true | 要求收盘 > MA60 长期均线 |
| touchPct | 0.01 | 昨日最低「贴近下轨」容差 |
| stopBufferPct | 0.015 | 收盘跌破下轨逾此比例→破位止损 |
| breakevenAfterTP1 | true | 中轨止盈后止损上移到成本 |
| maxHold | 40 | 最长持有根数（超时失效离场） |
| cooldown | 3 | 离场后冷却根数 |
